Using AI to Pentest My Own Code

كيف خليت الذكاء الاصطناعي يسوي اختبار اختراق للكود حقي

مارس ٢٠٢٦ March 2026 · 6 دقائق قراءة 6 min read · أمن سيبراني Security
Abdulaziz Alodan · عبدالعزيز العودان Cybersecurity Researcher & Full-Stack Developer · Saudi Arabia

Recently I tried something a bit unusual.

Instead of using traditional vulnerability scanners to test my application, I let an AI analyze the entire codebase and attempt a security review.

The result?
It discovered vulnerabilities I had not noticed before.

In this write-up, I explain how I used AI to perform a pentest-style analysis directly from the project source code.

قبل فترة جربت شيء مختلف شوي.

بدل ما أستخدم أدوات الفحص التقليدية عشان أختبر أمان التطبيق، خليت الذكاء الاصطناعي يقرأ المشروع كامل ويحاول يسوي تحليل أمني للكود.

والنتيجة؟
طلع ثغرات ما كنت منتبه لها.

في هذا الـ write-up أشرح كيف استخدمت AI عشان يسوي اختبار اختراق للتطبيق مباشرة من الكود.

01

The Idea

الفكرة

EN

The idea is simple.

Instead of testing the application only from the outside, I let AI read the full project and analyze the source code itself.

This makes the process closer to white-box security testing.

Because the AI can understand the internal logic of the application, it can detect issues that many traditional scanners miss.

AR

الفكرة بسيطة.

بدل ما أفحص التطبيق من الخارج فقط، خليت AI يقرأ المشروع كامل ويحلل الكود نفسه.

وبكذا صار الموضوع أقرب إلى white-box security testing.

ولأن AI يشوف منطق التطبيق الداخلي، صار يقدر يكتشف مشاكل كثير من أدوات الفحص التقليدية ما تلاحظها.

02

The Tool I Used

الأداة اللي استخدمتها

EN

For my test, I used Codex CLI.

Not because it is the only tool capable of doing this, but because it works well for analyzing codebases and understanding project structure.

I also chose not to use Claude Code in this experiment because I wanted to save my tokens for other tasks.

This same idea can also be done with:

  • Claude Code
  • Gemini CLI
  • Any AI tool that can read the full repository
AR

بالنسبة لي استخدمت Codex CLI.

مو لأنه الأداة الوحيدة اللي تقدر تسوي هذا الشيء، لكن لأنه مناسب لتحليل المشاريع ويفهم الكود وهيكلة المشروع بشكل كويس.

وبرضو اخترت ما أستخدم Claude Code في هالتجربة لأني كنت أبغى أوفر الـ tokens لمهام ثانية.

ونفس الفكرة تقدر تسويها باستخدام:

  • Claude Code
  • Gemini CLI
  • أي أداة AI تقدر تقرأ المشروع كامل
03

The Method

الطريقة

EN

The workflow was simple.

First, I opened the terminal inside the project folder.

Then I launched Codex CLI and gave it a direct prompt that clearly defined the scope and restrictions.

I wanted it to review the project like an authorized pentest, without modifying anything.

Prompt Used 
I own this app and I authorize you to pentest it.
DO NOT EDIT/CHANGE any code or config, no commits.

Method: First test OWASP Top 10 systematically,
then hunt deeper/advanced vulnerabilities.

Read the full project carefully, inspect routes, auth,
input handling, storage, APIs, business logic,
secrets, files, and dangerous flows.

You may run safe read-only tests and interact with
the running app if needed, but do not make
destructive changes.

Give me:
1. Confirmed vulnerabilities
2. High-confidence suspicious findings
3. Exploitation steps
4. Severity
5. Exact vulnerable files/functions
6. Fix recommendations

أول شيء فتحت التيرمنل داخل مجلد المشروع.

بعدها شغلت Codex CLI وأعطيته برومبت واضح يحدد النطاق والقيود بشكل مباشر.

كنت أبغاه يراجع المشروع كأنه pentest مصرح به، لكن بدون ما يعدل أي شيء.

04

Why This Works Better Than Normal Scanning

ليش هذه الطريقة أقوى من الفحص التقليدي

EN

Because the AI can see the full codebase, the process becomes more than normal external scanning.

It can combine two perspectives:

  • reading the code directly
  • interacting with the application behavior

That combination makes it much better at finding logic flaws, insecure flows, weak validation, and code-level mistakes.

In other words, it does not only scan blindly.
It reads, understands, tests, and connects the dots.

AR

لأن AI يشوف المشروع كامل، صار الموضوع أكبر من مجرد فحص خارجي تقليدي.

صار يجمع بين منظورين:

  • قراءة الكود مباشرة
  • تجربة سلوك التطبيق فعليًا

وهذا الدمج يخليه أفضل بكثير في اكتشاف مشاكل المنطق، والتدفقات غير الآمنة، وضعف التحقق، والأخطاء الموجودة في الكود نفسه.

بمعنى آخر، هو ما يفحص بشكل أعمى فقط.
هو يقرأ، ويفهم، ويجرب، ويربط الأمور ببعض.

05

What Happened During Testing

وش صار أثناء الاختبار

EN

During the test, the AI started probing endpoints, analyzing responses, and checking how the application behaved.

What I liked most was that it did not depend only on scanning.
It combined source-code understanding with application interaction.

That is exactly what made the findings more useful.

AR

أثناء الاختبار، بدأ AI يجرب على الـ endpoints، ويحلل الردود، ويشوف كيف التطبيق يتصرف.

والشيء اللي عجبني أكثر أنه ما اعتمد على scanning فقط.
جمع بين فهم الكود نفسه وبين التفاعل مع التطبيق.

وهذا بالضبط اللي خلّى النتائج أكثر فائدة.

06

Takeaway

الخلاصة العملية

EN

If you want a stronger security review of your own application, this approach is worth trying.

Running an AI agent from inside the project gives you a powerful way to review:

  • OWASP Top 10 issues
  • business logic flaws
  • weak validation
  • insecure implementation details
  • suspicious attack paths

Directly from the terminal.

AR

إذا كنت تبغى مراجعة أمنية أقوى لتطبيقك، فهذي الطريقة تستحق التجربة.

تشغيل AI agent من داخل المشروع يعطيك طريقة قوية لمراجعة:

  • مشاكل OWASP Top 10
  • ثغرات منطق العمل
  • ضعف التحقق
  • التفاصيل البرمجية غير الآمنة
  • المسارات الهجومية المحتملة

وكل هذا مباشرة من التيرمنل.

Conclusion

This experiment showed me that AI can be more than just a coding assistant.

When used correctly, it can act like a practical security reviewer that reads code, tests behavior, and helps uncover vulnerabilities that might otherwise be missed.

الخاتمة

هالتجربة بينت لي أن AI يقدر يكون أكثر من مجرد مساعد برمجي.

إذا استخدمته بشكل صحيح، يقدر يشتغل كمراجع أمني عملي يقرأ الكود، ويختبر السلوك، ويساعدك تطلع ثغرات ممكن ما تنتبه لها بالطريقة المعتادة.
